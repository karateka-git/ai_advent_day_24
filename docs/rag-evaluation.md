# Evaluation RAG-режимов

Документ описывает текущий формат evaluation после добавления второго этапа retrieval:

- `none`
- `threshold-filter`
- `heuristic-filter`
- `heuristic-rerank`
- `model-rerank`

`README.md` на этом этапе намеренно не менялся. Этот документ нужен как краткое описание того, как теперь устроен evaluation и какие артефакты он создаёт.

## Цель

Evaluation должен отвечать на два вопроса:

- как разные post-retrieval режимы ведут себя на каждом контрольном вопросе;
- какой режим чаще даёт лучший результат по корпусу в целом.

Основной акцент сделан не на длинных полных ответах, а на краткой сводке по каждому вопросу:

- сколько RAG-режимов справились;
- какие режимы оказались лучшими;
- где фильтрация помогла убрать шум;
- где rerank поднял правильный фрагмент выше.

## Скрипт

Основной сценарий:

- [run-rag-evaluation.ps1](/C:/Users/compadre/Downloads/Projects/AiAdvent/day_24/scripts/run-rag-evaluation.ps1)

Скрипт делает следующее:

1. Индексирует корпус для выбранной стратегии.
2. Для каждого вопроса запускает:
   - `plain`
   - `rag + none`
   - `rag + threshold-filter`
   - `rag + heuristic-filter`
   - `rag + heuristic-rerank`
   - `rag + model-rerank`
3. Из каждого запуска извлекает:
   - answer body
   - retrieval summary
4. Автоматически присваивает verdict:
   - `success`
   - `partial`
   - `fail`
5. Формирует два markdown-артефакта:
   - краткий summary report
   - raw report для подробного разбора

## Артефакты

По умолчанию скрипт записывает:

- `data/rag-evaluation-summary.md`
- `data/rag-evaluation-raw.md`

### Summary report

Это основной отчёт для чтения человеком.

Для каждого вопроса он содержит:

- формулировку вопроса;
- ожидание и ожидаемые источники;
- краткую таблицу по режимам:
  - режим
  - verdict
  - score
  - краткая заметка;
- итог по вопросу:
  - сколько RAG-режимов дали `success`;
  - сколько дали `partial`;
  - какие режимы оказались лучшими;
  - короткий вывод.

### Raw report

Это отчёт для отладки и детального анализа.

Для каждого вопроса он хранит:

- полный answer body по каждому режиму;
- retrieval summary по каждому режиму;
- verdict и score;
- note с кратким пояснением.

Все RAG-запуски в raw report выполняются с `--show-all-candidates`, поэтому отчёт подходит для разбора retrieval pipeline.

## Как интерпретировать verdict

Сейчас evaluation использует три статуса:

- `success`
- `partial`
- `fail`

Оценка эвристическая, а не претендует на строгую академическую метрику. Скрипт учитывает:

- попал ли ожидаемый источник в retrieval;
- насколько термины ответа пересекаются с ожидаемыми фактами;
- похож ли ответ на честный отказ из-за отсутствия контекста.

Этого достаточно для учебного сравнения режимов и для быстрого выявления вопросов, которые стоит проверить вручную.

## Как читать отчёт

Рекомендуемый порядок:

1. Посмотреть, сколько RAG-режимов дали `success`.
2. Посмотреть, кто оказался лучшим по `score`.
3. Прочитать краткие заметки в summary.
4. Если вопрос спорный, открыть raw report и посмотреть retrieval summary для всех режимов.

Summary отвечает на вопрос:

- какой режим лучше сработал на конкретном вопросе.

Raw report помогает понять:

- почему именно это произошло.

## Как запускать

1. Собрать проект:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\start-manual-check.ps1
```

2. Запустить evaluation:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\run-rag-evaluation.ps1
```

3. Открыть:

- `data/rag-evaluation-summary.md`
- `data/rag-evaluation-raw.md`

4. Если нужно, перепроверить отдельные вопросы вручную через CLI с `--post-mode` и `--show-all-candidates`.

## Практический смысл этапа

После внедрения post-retrieval режимов evaluation нужен для трёх вещей:

- показать разницу между baseline retrieval и retrieval с фильтрацией/реранкингом;
- увидеть, где rule-based режимы уже достаточно хороши;
- понять, даёт ли `model-rerank` реальный выигрыш относительно более дешёвых эвристик.

Именно это и является итогом текущего этапа: не только реализовать несколько retrieval-режимов, но и сделать их сравнение воспроизводимым и удобным для чтения.
